credit_card_picker · data curation

Card entry form

Fill this out to generate a schema-valid card file for data/cards/. Paste each source link at the moment you enter the fact, not after — the checks panel mirrors scripts/validate_cards.py. Full guidance: docs/curation-guide.md.

Identity

The id becomes the filename and the issuer becomes the directory — the validator enforces both, so files can't drift from their paths.

Earning & base_rate

Cash cards: rate = percent. Points cards: rate = points per dollar, priced via the shared valuation table — cards never embed their own point values. The base rate is the catch-all every reward is an exception to.

category_rewards

Spend-proportional elevated rates. Caps are on spend, not reward ("$300 back at 6%" = cap $5,000). Use the rotating pseudo-category for quarterly cards; never hardcode this quarter's categories.

merchant_rewards

Only when the issuer names a merchant ("5% at Amazon") — merchant beats category when both match.

credits

Fixed-value, use-it-or-lose-it, never scaled by spend. The capture-rate note is required: issuers price annual fees against sticker credit value most users never capture — say honestly who gets it and who doesn't.

signup_bonus

One-time, year-one only, current public offer only. Mark limited-time elevated promos and their end date — the validator nags once an offer expires so it can't silently inflate recommendations.

fees

The costs every reward is netted against.

approval

So no card is ever recommended that the user can't get. Cutoffs aren't published — estimate from the issuer's own "recommended credit" language and say why in notes.

closed_loop store cards only

Marks cards usable only at specific merchants. Expresses the restriction — the reward still lives in base_rate / merchant_rewards.

benefit_flags

Binary perks, recorded but unscored in v1. Reuse existing spellings (grep other cards) before inventing new ones.

sources

The paper trail. Paste the exact page you read — issuer pages are primary, blogs are corroboration only. Mark honestly which blocks each link backs: a source showing only the annual fee supports fees, not everything.

verification

Sources say where facts came from; verification says when a human last confirmed the whole file still matches them. Drafting without checking sources = confidence low.
